# 2026-07-07 项目骨架搭建

## 做了什么

### 后端 (Python/FastAPI)
- 搭建完整目录结构：API / Agent / Collector / Analyzer / Notifier / Repository / Models
- 实现 10 个 REST API 端点：chat, trends, topics/hot, dashboard, reports/latest, reports/generate 等
- 实现 LangGraph Agent 工作流：意图识别 → 任务规划 → 工具调用 → 回复生成
- 实现 4 个平台数据采集器：GitHub (API)、B站/小红书/知乎 (MediaCrawler + Mock 降级)
- 实现 DataSource 抽象接口，各平台解耦可替换
- 实现热度标准化算法（对数变换 + 时间衰减 + 平台权重）
- 实现定时任务引擎 (APScheduler)：采集(6h) + 日报生成(每日09:00) + 突发检测(1h)
- 实现飞书 Bot 推送：Markdown 日报 + 文本告警 + @所有人
- 7 张数据库表：raw_posts, daily_hotness, monitored_topics, conversation_history, daily_reports, feishu_config, alert_subscriptions

### 前端 (Vue 3 + TypeScript)
- 3 个页面：仪表盘、对话、日报
- 4 个组件：SearchBar, TrendChart(ECharts), HotTopicCard, PostList
- Pinia 状态管理 + Vue Router + Axios API 封装

### 基础设施
- Docker Compose (backend + web)
- Makefile
- .gitignore

## 技术决策
- Python 3.9 兼容（降级 numpy < 2.0，使用 `from __future__ import annotations`）
- Agent 依赖通过闭包注入（避免 msgpack 序列化问题）
- 采集器架构：各平台独立实现 DataSource 接口，通过 Supervisor 统一调度
- Mock 降级：MediaCrawler 不可用时自动使用 Mock 数据，保证开发流程不中断

- `/api/dashboard` ✅ 返回结构化数据
